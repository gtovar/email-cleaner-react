# Format rules
- Each entry is a short factual event.
- No user stories. No backlog. No planning.
- No repetition of Project_State.md.
- No narratives or explanations.
- No opinions or speculation.
- No technical debates or code details.
- Only record events that *actually happened*.
- Always in English.
- Chronological order: newest last.
- 1–4 lines per entry.
